How to implement it

Let's say you are tasked to combine abstract classes, interfaces, and multipla interfaces to create a program where vehicles can start, stop, and display their fuel type. Also, some vehicles may have additional capabilities, like GPS.

Here's how you can implement it:


// Interface for basic vehicle operations
interface Vehicle {{'{'}}
    void start();
    void stop();
{{'}'}}

// Interface for GPS navigation
interface GPS {{'{'}}
    void navigate();
{{'}'}}

// Abstract class for shared functionality
abstract class FuelVehicle implements Vehicle {{'{'}}
    abstract void fuelType(); // Abstract method for fuel type
    
    public void start() {{'{'}}
        System.out.println("Vehicle is starting");
    {{'}'}}

    public void stop() {{'{'}}
        System.out.println("Vehicle is stopping");
    {{'}'}}
{{'}'}}

// Car class implementing both interfaces
class Car extends FuelVehicle implements GPS {{'{'}}
    public void fuelType() {{'{'}}
        System.out.println("Car uses petrol");
    {{'}'}}

    public void navigate() {{'{'}}
        System.out.println("Car navigation system is on");
    {{'}'}}
{{'}'}}

// Bike class only implements FuelVehicle
class Bike extends FuelVehicle {{'{'}}
    public void fuelType() {{'{'}}
        System.out.println("Bike uses diesel");
    {{'}'}}
{{'}'}}

// Main class to test the implementation
public class Main {{'{'}}
    public static void main(String[] args) {{'{'}}
        FuelVehicle myCar = new Car();
        FuelVehicle myBike = new Bike();

        System.out.println("Car:");
        myCar.start();
        myCar.fuelType();
        ((Car) myCar).navigate(); // Type casting to access GPS
        myCar.stop();

        System.out.println("\nBike:");
        myBike.start();
        myBike.fuelType();
        myBike.stop();
    {{'}'}}
{{'}'}}

Best Practices

When implementing abstract classes and interfaces, follow these best practices:

Use abstract classes for shared functionality with variation.

Use interfaces to define contracts for unrelated classes.

Common Use Cases

Abstract classes and interfaces are used in many real-world scenarios because they help in creating a clear separation between the internal workings of an object and the external interface, leading to more robust, maintainable, and secure code.

Here are some examples:

Interfaces

When enforcing rules, like ensuring all classes that implement Playable have a play() method.

Abstract Classes

When building a base class for similar objects, like a Vehicle class for Car, Bike, etc.
