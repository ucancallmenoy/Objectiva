Basic class structure

Understanding the structure of classes and how they interact with access modifiers, instance/static members, and methods is essential in writing robust Java programs.

A class in Java serves as a blueprint for creating objects, encapsulating data (fields) and behaviors (methods). Classes are the foundation of object-oriented programming in Java.

Fields

It represents the state or properties of an object. For example, a class “Car” might have fields like “color”, “make”, and “speed”.


public class Car {{'{'}}
    String color; // Field to store the car's color
    int speed;    // Field to store the car's speed>
{{'}'}}

Constructors

It is a special method that initializes the fields of a class. Constructors are called automatically when creating an object.


// Constructor to initialize the fields
public Car(String color, int speed) {{'{'}}
    this.color = color;
    this.speed = speed;
{{'}'}}

Methods

Methods define behaviors or actions that the object can perform. For example, a “Car” class might have methods like “drive()” or “stop()”.


public class Car {{'{'}}
    public void drive() {{'{'}}
        System.out.println("Car is driving");
    {{'}'}}
{{'}'}}

Example of Class

Let's put it all together in a complete example of a Car class.


public class Car {{'{'}}
    // Fields
    String color;
    int speed;

    // Constructor
    public Car(String color, int speed) {{'{'}}
        this.color = color;
        this.speed = speed;
    {{'}'}}

    // Method
    public void drive() {{'{'}}
        System.out.println("The " + color + " car is driving at " + speed + " mph");
    {{'}'}}
{{'}'}}

Instance vs Static Members

Java classes can have instance members and static members, each serving distinct purposes.

Instance Members

Instance members belong to an instance of the class. Each object has its own copy of instance variables. Before we define our instance member, we should know class are references types, a variable of a class object holds a reference to the address of the object.


public class Car {{'{'}}
    // Instance variables (members)
    String color;   // Each car objects can have a different color
    int speed;      // Each car objects can have a different speed

    // Constructor to initialize instance variables
    public Car(String color, int speed){{'{'}}
        this.color = color;   // Assigning color value to the instance variable
        this.speed = speed;   // Assigning speed value to the instance variable
    {{'}'}}

    // Instance method
    public void drive() {{'{'}}
        // Instance methods belong to the objects and can access instance variables
        System.out.println("The " + color + " car is driving at " + speed + " mph");
    {{'}'}}
{{'}'}}
                
public class Main {{'{'}}
    public static void main(String[] args) {{'{'}}
        // Creating instance of the Car class with unique attributes
        // Creates a car object with color Red and speed 60
        Car car1 = new Car("Red", 60);
        // Creates a car object with color Blue and speed 80
        Car car2 = new Car("Blue", 80);  

        // Calling the instance method for each car object
        car1.drive();
        car2.drive();
    {{'}'}}
{{'}'}}

Result:

The Red car is driving at 60 mph

The Blue car is driving at 80 mph

Static Members

Static members belong to the class rather than any instance. It is shared among all objects of the class. A static member can be accessed through the class itself, even if no instance of the class exists. Static members are always referenced by the class name, not by an instance.


public class Car {{'{'}}
    // Static variable (member)
    static int totalCars = 0;   // Tracks the total number of car instances

    // Constructor to increment the static variable whenever a new object is created
    public Car() {{'{'}}
        totalCars++;    // Increment the static variable each time a Car object is created
    {{'}'}}

    // Static method
    public static void displayTotalCars() {{'{'}}
        // Static method can accesss static members directly
        System.out.println("Total number of cars: " + totalCars);
    {{'}'}}
{{'}'}}
                
public class Main {{'{'}}
    public static void main(String[] args) {{'{'}}
        // Creating instances of the Car class
        // Creates a car object, increments totalCars by 1
        Car car1 = new Car();   
        // Creates a car object, increments totalCars by 2
        Car car2 = new Car();   

        // Calling static method on the class (no need for an instance)
        Car.displayTotalCars();
    {{'}'}}
{{'}'}}

Result:

Total numbers of cars: 2

Method Parameters and Return Types

Method parameters are the values you give to a method when you call it. They act like “inputs” for the method, and the method uses these inputs to do something or calculate something. You define parameters in the method, and when you call the method, you give it the actual values called arguments. You can add as many parameters as you want, as long it has a comma to separate them.

Imagine you're at a pizza shop, and you're placing an order. You have two choices to make: the size of the pizza and the topping you want.


public class PizzaShop {{'{'}}
    // The method has two parameters: size and topping
    public void orderPizza(String size, String topping) {{'{'}}
        System.out.println("You ordered a " + size + " pizza with " + topping + " topping.");
    {{'}'}}
                
    public static void main(String[] args) {{'{'}}
        // Create a PizzaShop object
        PizzaShop myPizzaShop = new PizzaShop(); 
        // Call the method with arguments
        myPizzaShop.orderPizza("Large", "Pepperoni"); 
        // Call the method with different arguments
        myPizzaShop.orderPizza("Medium", "Mushroom"); 
    {{'}'}}
{{'}'}}

Result:

You ordered a Large pizza with Pepperoni topping.

You ordered a Medium pizza with Mushroom topping.

Return Types

Return types tell you what kind of value a method will give back after it finishes. If a method does not need to return anything, we use “void”. If the method needs to give back a value like a number or text, we specify the type of that value like “int” for integers or “double for decimal numbers.

Example with No Return Value (Void)

Imagine you're sending a text message to a friend. You don’t need a response; you’re just informing them about something.


public class TextMessage {{'{'}}
    // No return value, just sends a message
    public void sendMessage(String message) {{'{'}}
        System.out.println("Message sent: " + message);
    {{'}'}}
                
    public static void main(String[] args) {{'{'}}
        // Create a TextMessage object
        TextMessage myMessage = new TextMessage(); 
        // Call the method to send a message
        myMessage.sendMessage("Hey! I'll meet you at 5 PM."); 
    {{'}'}}
{{'}'}}

Result:

Message sent: Hey! I'll meet you at 5 PM.

Example with Return Value

Imagine you're checking the weather app to know the temperature. You need the app to return the current temperature to you.


public class WeatherApp {{'{'}}
    // Returns the current temperature
    public int getTemperature() {{'{'}}
        return 75; // Return the temperature (in Fahrenheit)
    {{'}'}}
                
    public static void main(String[] args) {{'{'}}
        // Create a WeatherApp object
        WeatherApp myWeatherApp = new WeatherApp(); 
        // Store the returned value
        int temperature = myWeatherApp.getTemperature(); 
        // Display the result
        System.out.println("The current temperature is " + temperature + "°F."); 
    {{'}'}}
{{'}'}}

Result:

The current temperature is 75°F.
