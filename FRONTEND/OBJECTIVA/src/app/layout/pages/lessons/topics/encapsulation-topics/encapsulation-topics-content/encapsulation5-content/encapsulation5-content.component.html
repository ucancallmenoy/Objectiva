Data Validation

Let's start by understanding why data validation is crucial in encapsulation.

Think of data validation as a security guard and quality control inspector combined - it ensures that only valid data enters our encapsulated classes and that the object always remains in a consistent state.

Validating user inputs

User input validation is like having a security guard at the entrance of a building. Before any data enters our system, we need to verify that it meets our requirements and won't cause problems.

There are several layers of validation we need to consider:

Type Validation

Ensuring the data is of the correct type

Format Validation

Checking if the data follows required patterns

Range Validation

Verifying values fall within acceptable limits

Business Rule Validation

Ensuring data meets specific business requirements

Here's a simple example with a Student class:

Preventing invalid states

Preventing invalid states is about maintaining the integrity of our objects throughout their lifecycle. It's like maintaining a car - we need to ensure all parts work together correctly and stay in a valid state.

Sometimes we need to make sure our object stays in a valid state - like making sure a bank account doesn't go negative. Here's a simple example:

Common practices for validation in Java

Here's a simple example showing common ways to validate data:

This program demonstrates encapsulation by creating a `Product` class with private fields for name, price, and quantity. The program prevents invalid states such as empty names, negative prices, and quantities exceeding specified limits.


public class Product {{'{'}}
    private String name;
    private double price;
    private int quantity;

    // Constructor with validation
    public Product(String name, double price, int quantity) {{'{'}}
        // Validate all inputs before creating object
        validateName(name);
        validatePrice(price);

        // If all valid, save the data

    // Simple validation methods

            throw new IllegalArgumentException("Name is too long");
        {{'}'}}
    {{'}'}}

    private void validatePrice(double price) {{'{'}}
