What are interfaces?

An interface in Java is like a list of rules or tasks that a class must follow. Itâ€™s similar to a class but only contains method names (without actual code). When a class uses an interface, it agrees to provide its own code for all the methods in the interface.

Interface implementation

To use an interface, a class adds the word implements in its definition. This means the class is agreeing to follow the rules and provide code for all the methods listed in the interface.

Multiple Interfaces

In Java, multiple interfaces refer to the ability of a class or interface to implement or extend more than one interface.

Now, how to implement these type of interfaces?

An interface is like a contract that specifies what a class must do, but not how it does it.


interface Vehicle {{'{'}}
    void start(); // Method without a body
    void stop();
{{'}'}}

A class can implement an interface and provide the body for its methods.


                class Car implements Vehicle {{'{'}}
    public void start() {{'{'}}
        System.out.println("Car is starting");
    {{'}'}}
    public void stop() {{'{'}}
        System.out.println("Car is stopping");
    {{'}'}}
{{'}'}}

A class can implement multiple interfaces.


interface Camera {{'{'}}
    void takePhoto();
{{'}'}}
interface GPS {{'{'}}
    void navigate();
{{'}'}}
class Smartphone implements Camera, GPS {{'{'}}
    public void takePhoto() {{'{'}}
        System.out.println("Photo taken.");
    {{'}'}}
    public void navigate() {{'{'}}
        System.out.println("Navigating to destination.");
    {{'}'}}
{{'}'}}

Interfaces vs Abstract Class

Interfaces are used when you want to define what a class should do, but not how it does it. Abstract classes are used when you want to define common behavior that multiple subclasses can share.

But when to use which?

Interface

When defining a contract for unrelated classes or enabling multiple inheritance

Abstract Class

When creating a base for related classes with shared behavior or fields
