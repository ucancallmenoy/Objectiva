Encapsulation syntax

Encapsulation in Java is like wrapping your data in a protective package. Think of it as creating a secure container for your information, where you control how that information can be accessed and modified.

The basic syntax follows this pattern:


public class Student {{'{'}}
    // Private data (hidden from outside)
    private String name;
    private int age;

    // Public methods to access and modify the data
    public void setName(String name) {{'{'}}
        this.name = name;
    {{'}'}}

    public String getName() {{'{'}}
        return name;
    {{'}'}}
{{'}'}}

Examples of encapsulated classes

Let's look at a simple bank account example to understand how encapsulation works in practice:


public class BankAccount {{'{'}}
    // Private data members
    private double balance;
    private String accountNumber;

    // Public constructor
    public BankAccount(String accountNumber) {{'{'}}
        this.accountNumber = accountNumber;
        this.balance = 0.0;
    {{'}'}}

    // Public methods for controlled access
    public void deposit(double amount) {{'{'}}
        if (amount > 0) {{'{'}}
            balance += amount;
        {{'}'}}
    {{'}'}}

    public double getBalance() {{'{'}}
        return balance;
    {{'}'}}
{{'}'}}

In this example, we've encapsulated the account details so that the balance can't be changed directly, deposits must go through a controlled method, and account information is protected but accessible.

Best practices

When implementing encapsulation, follow these important guidelines:

Keep Fields Private

Make all instance variables private unless there's a very good reason not to. This prevents unauthorized access.


private String name;
private int age;

Use Meaningful Names
