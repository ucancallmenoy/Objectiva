What is abstraction?

It is the process of concealing implementation details while exposing only the necessary functionality to the user. In other words, it highlights essential features while hiding internal complexities.

It is like hiding your true feelings behind a mask so that you can't show your emotions inside.

Levels of Abstraction

There are three levels of abstraction:

High-Level Abstraction

Deals with the "what" of the system, focusing on user interaction and overall system behavior.

Mid-Level Abstraction

Involves system components and their responsibilities, typically focusing on subsystems or modules.

Low-Level Abstraction

Details the "how," dealing with specific implementation, such as code and algorithms.

But where does these levels of abstractions are commonly used?

High-Level Abstraction

Typically, interfaces or abstract classes represents high-level abstractions. These define contracts or templates that classes must follow, but they do not specify how those contracts are carried out.

Example: An interface like Shade with methods like draw() and area(). The interface defines what actions a shape can perform, but the specifics of how it's done are left to the implementing classes

Mid-Level Abstraction

Abstract classes provide this level of abstraction. They can have both abstract and concrete methods, offering a mix of shared functionality and customization

An abstract class Animal may provide breathe() method (concrete) but leave the makeSound() method abstract to be implemented by subclasses like Dog or Cat

Low-Level Abstraction

Concrete classes implement low-level abstraction, where all methods are defined, and the actual behavior is fully specified.

A Dog class that extends Animal and provides a specific implementation of makeSound().

What are the benefits of Abstraction?

Abstraction provides several benefits:

Simplifies Complexity

Focuses on essential details, hiding unnecessary ones for easier understanding

Enahances Reusability

Abstract classes and interfaces provide reusable and extendable code structures

Improves Maintainability

Changes in implementation don't affect high-level abstractions, making updates easier

Encourages Modularity

Breaks down systems into manageable, independent components

Supports Scalability

Allows adding new functionality without altering existing code significantly
