Compile-time Polymorphism

Compile-time polymorphism means deciding which function or method to use at compile time, before the program runs. It's like giving the program multiple tools (methods) to do the same task, but which tool to use is decided early, based on the inputs or context.

It is achieved through method overloading or operator overloading.

Method Overloading

Method overloading occurs when two or more methods in the same class have the same name but different parameter lists (in terms of number, type, or order of parameters).

It's like a coffee machine that can make coffee with just a button press, or lets you specify the type of coffee or size depending on your choice.

How about in coding?


class Calculator {{'{'}}
    // Overloaded methods
    int add(int a, int b) {{'{'}}
        return a + b;
    {{'}'}}

    double add(double a, double b) {{'{'}}
        return a + b;
    {{'}'}}

    int add(int a, int b, int c) {{'{'}}
        return a + b + c;
    {{'}'}}
{{'}'}}

public class Main {{'{'}}
    public static void main(String[] args) {{'{'}}
        Calculator calc = new Calculator();
        System.out.println(calc.add(5, 10));        // Output: 15
        System.out.println(calc.add(5.5, 10.5));    // Output: 16.0
        System.out.println(calc.add(5, 10, 15));    // Output: 30

Constructor Overloading

Constructor overloading happens when a class has multiple constructors with the same name but different parameter lists. This allows the creation of objects in different ways based on the given arguments.

It's like a hotel booking system where you can book a room with just a name, or add details like the number of nights, or even include meal preferences.

How about in coding?


class Person {{'{'}}
    String name;
    int age;

    // Default constructor
    Person() {{'{'}}
        this.name = "John Doe";
        this.age = 30;
    {{'}'}}

    Constructor with one parameter
    Person(String name) {{'{'}}
        this.name = name;
        this.age = 30;
    {{'}'}}

    Constructor with two parameters
    Person(String name, int age) {{'{'}}
        this.name = name;
        this.age = age;
    {{'}'}}

    void display() {{'{'}}
        System.out.println("Name: " + name + ", Age: " + age);
    {{'}'}}
{{'}'}}

public class Main {{'{'}}
    public static void main(String[] args) {{'{'}}
        Person person1 = new Person();
        Person person2 = new Person("Alice");
        Person person3 = new Person("Bob", 25);

        person1.display();    // Output: Name: John Doe, Age: 30
        person2.display();    // Output: Name: Alice, Age: 30
        person3.display();    // Output: Name: Bob, Age: 25
    {{'}'}}
{{'}'}}

Operator Overloading

Although Java does not support operator overloading explicitly like some other languages (e.g., C++), it can be mimicked by redefining the behavior of methods. However, the + operator is overloaded for string concatenation.

It's like the "+" symbol, which can add numbers (2 + 3 = 5) or combine words ("Hello" + "World" = "HelloWorld").

How about in coding?


public class Main {{'{'}}
    public static void main(String[] args) {{'{'}}
        '+' operator overloaded for numbers and strings
        int a = 10, b = 20;
        String str1 = "Hello", str2 = "World";

        System.out.println(a + b);        // Output: 30
        System.out.println(str1 + str2);  // Output: HelloWorld
    {{'}'}}
{{'}'}}

Type Promotion

Type promotion occurs when a smaller data type is automatically converted to a larger data type during method calls. This allows overloaded methods to accept parameters of varying types.

It's like a chef who can cook a dish with just the main ingredient, or add more ingredients to make it tastier.

How about in coding?


class TypePromotionDemo {{'{'}}
    void display(int a) {{'{'}}
        System.out.println("Integer: " + a);
    {{'}'}}

    void display(double a) {{'{'}}
        System.out.println("Double: " + a);
    {{'}'}}
{{'}'}}

public class Main {{'{'}}
    public static void main(String[] args) {{'{'}}
        TypePromotionDemo demo = new TypePromotionDemo();

        demo.display(10);       // Promotes int to match the int method
        demo.display(10.5);     // Matches the double method directly
